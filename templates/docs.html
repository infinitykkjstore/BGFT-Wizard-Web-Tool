Sessions use cookies. Add -c cookies.txt -b cookies.txt to commands for persistent session logs.

GET /api/status

Method: GET | Endpoint: {{ host }}/api/status

Returns server setup state.

  • ready: boolean, true when builder environment is ready.
  • error: string or null, setup error if initialization failed.
curl "{{ host }}/api/status"

Response example

{"ready":true,"error":null}

GET /api/logs

Method: GET | Endpoint: {{ host }}/api/logs

Returns user-visible logs for the current session only.

  • Requires same session cookie to keep continuity.
  • Server/internal logs are not exposed here.
curl -c cookies.txt -b cookies.txt "{{ host }}/api/logs"

Response example

{"logs":["[12:32:10] Extracting metadata from: https://example.com/game.pkg"]}

GET /api/meta

Method: GET | Endpoint: {{ host }}/api/meta

Extracts PKG/manifest metadata.

  • url (required): direct PKG URL or manifest URL.
curl -c cookies.txt -b cookies.txt --get "{{ host }}/api/meta" --data-urlencode "url=https://example.com/game.pkg"

Success response fields

  • success: boolean
  • title: package title
  • title_id: CUSA-style id when available
  • content_id: full content id when available
  • category: category string (example: gd)
  • pkg_size: integer bytes
  • pkg_type: BGFT type (example: PS4GD)
  • icon_path: icon endpoint URL or null

GET /api/build

Method: GET | Endpoint: {{ host }}/api/build

Builds payload.bin and stores it in temp output with random filename.

Query parameters

  • url (required): PKG/manifest URL.
  • name (required): package name/title.
  • id (required): title/content id.
  • icon (optional): icon URL.
  • type (optional): default PS4GD.
  • size (optional): package size in bytes (integer).
curl -c cookies.txt -b cookies.txt --get "{{ host }}/api/build" --data-urlencode "url=https://example.com/game.pkg" --data-urlencode "name=My Game" --data-urlencode "id=CUSA00000" --data-urlencode "icon={{ host }}/api/icon/sample.png" --data-urlencode "type=PS4GD" --data-urlencode "size=1234567890"

Success response fields

  • success: boolean
  • file: generated filename for download
  • size: payload size in bytes
  • logs: session log lines related to build

Error responses

  • 400: missing params or environment not ready
  • 500: compile/runtime failure, includes error

GET /api/download/<filename>

Method: GET | Endpoint: {{ host }}/api/download/<file>

Downloads generated payload file as payload.bin.

curl -L "{{ host }}/api/download/REPLACE_WITH_FILE.bin" -o payload.bin

GET /api/icon/<filename>

Method: GET | Endpoint: {{ host }}/api/icon/<file>

Returns extracted icon as PNG when available.

curl -L "{{ host }}/api/icon/REPLACE_WITH_ICON.png" -o icon.png

POST /api/cleanup

Method: POST | Endpoint: {{ host }}/api/cleanup

Removes stale payload output files from temp directory.

curl -X POST "{{ host }}/api/cleanup"

Response example

{"success":true}
